# Artefatos Visuais do NotebookLM

Esta pasta/documentacao foi criada para registrar os artefatos gerados no Estudio do NotebookLM. Eles ajudam a demonstrar visualmente o uso da ferramenta no desafio da DIO.

## Artefatos recomendados

### 1. Mapa mental

Uso recomendado: mostrar a organizacao dos principais conceitos estudados.

Por que incluir: e o artefato visual mais forte para comprovar que o tema foi estruturado em blocos de conhecimento.

Sugestao de arquivo:

```text
assets/notebooklm/mapa-mental.png
```

### 2. Infografico

Uso recomendado: apresentar visualmente o fluxo do estudo, ligando bolsa de valores, analise fundamentalista, dividendos, long-short e simulacao multiagente.

Por que incluir: deixa o repositorio mais visual e facilita a leitura para avaliadores.

Sugestao de arquivo:

```text
assets/notebooklm/infografico.png
```

Arquivo incluido no projeto:

![Infografico sobre arquitetura de um mercado financeiro artificial](../assets/notebooklm/infografico-mercado-financeiro-artificial.png)

### 3. Cartoes didaticos

Uso recomendado: revisar conceitos como CVM, B3, dividend yield, margem de seguranca, long-short, market neutral e risco de mercado.

Por que incluir: mostra que o projeto tambem gerou material de revisao.

Sugestao de arquivo:

```text
assets/notebooklm/cartoes-didaticos.pdf
```

### 4. Teste

Uso recomendado: gerar perguntas de verificacao de aprendizagem.

Por que incluir: reforca o uso da IA como ferramenta de aprendizagem ativa.

Sugestao de arquivo:

```text
assets/notebooklm/teste-revisao.pdf
```

## Artefatos opcionais

- **Apresentacao de slides**: boa para transformar o projeto em uma apresentacao curta.
- **Resumo em audio**: interessante, mas menos pratico para avaliacao no GitHub.
- **Resumo em video**: bom se o NotebookLM permitir exportar ou compartilhar.
- **Relatorios**: util caso a resposta gerada seja mais completa que o miniguia.
- **Tabela de dados**: opcional, faria sentido se houver comparacao entre fontes, conceitos ou estrategias.

## Como documentar no repositorio

Depois de gerar os artefatos no NotebookLM:

1. Salve prints, PDFs ou arquivos exportados na pasta `assets/notebooklm`.
2. Atualize o `README.md` com links para esses arquivos.
3. Use os artefatos como evidencias visuais do processo de estudo.
